## 3-1. 첫 번째 개선안 상세 (공통 적용)

### 제안명

Intent + Review + Cleanup 루프

### 프로세스

1. 결과물 생성 시, 각 핵심 변경 옆에 `의도/판단 근거`를 짧게 표시한다.
2. 사용자에게 리뷰를 요청한다.
3. 리뷰 반영이 끝나면, 최종 산출물에서 임시 해설 문구를 제거해 깔끔한 본문만 남긴다.

### 기대 효과

- 리뷰어가 빠르게 맥락을 파악해 피드백 품질이 올라감
- 최종 문서는 군더더기 없는 형태로 유지됨
- 작성자/리뷰어 간 오해가 줄어듦

### 적용 방식 비교

- 옵션 A: 기존 모든 스킬의 공통 규칙으로 내장
- 옵션 B: 별도 스킬(예: `explain-then-cleanup`)로 분리

### 권장안

옵션 A를 먼저 적용하고, 반복 사용 패턴이 확정되면 옵션 B를 추가한다.

이유:
- 지금 문제는 "모든 스킬에서 일관되게 필요"한 성격이 강함
- 별도 스킬로 분리하면 사용자가 매번 호출 여부를 판단해야 해 누락 가능성이 큼
- 공통 규칙으로 정착시킨 뒤, 자동화 필요가 커질 때 분리하는 것이 운영 비용이 낮음

## 3-2. 두 번째 개선안 상세 (인지 에너지 보호)

### 문제 정의

AI 산출물 생성-리뷰 루프가 매우 빨라지면서, 사용자 인지 에너지가 빠르게 소모됨.
특히 애자일 계획 문서는 품질 판단 기준이 모호해 리뷰 피로가 커짐.

### 제안명

Pre-Review Quick Guide (문서 읽기 전 30초 가이드)

### 프로세스

1. AI는 결과물 본문 전에 `짧은 리뷰 가이드`를 먼저 제시한다.
2. 사용자는 가이드의 체크포인트만 먼저 확인한다.
3. 기준 충족 여부를 본 뒤 문서 본문을 읽는다.
4. 리뷰 완료 후, 필요 시 가이드는 요약만 남기거나 제거한다.

### 리뷰 가이드 표준 포맷 (권장)

- 이 문서의 목적(1문장)
- 이번에 확인할 것 3가지(범위, 우선순위, 실행 가능성)
- 위험 신호 2가지(예: 모호한 완료 조건, 과도한 태스크 분해)
- 5분 리뷰 순서(위에서 아래로 3-step)

### 기대 효과

- 문서 품질 판단 기준을 먼저 고정해 의사결정 피로 감소
- 비전문 영역 문서(애자일 계획 등) 리뷰 속도/정확도 개선
- "무엇을 봐야 하는지" 고민 시간을 최소화

### 운영 규칙 초안

- 모든 계획 문서는 본문 앞에 `Quick Guide`를 기본 포함
- 가이드는 최대 8줄 이내로 제한
- 용어는 추상 표현보다 검증 가능한 문장으로 작성

## 3-3. 중복 제거 결정 (US Summary 정리)

### 결정

`Quick Guide`가 공통 자동 생성된다면, 동일 역할의 `US Summary`는 제거한다.

### 제거 기준

- 목적이 "결과물 이해 보조"로 `Quick Guide`와 동일한 경우 제거
- Step/Phase 고유 결과 기록(증적, 메트릭, 결정 로그)이면 유지

### 마이그레이션 순서

1. `sprint-start`에서 `US Summary` 용어를 `Step Summary`로 임시 변경
2. `Step Summary` 내용을 `Quick Guide` 필드로 매핑
3. `Quick Guide` 자동 생성이 안정화되면 `Step/US Summary` 섹션 제거
4. 최종적으로 문서에는 본문 + 필요 증적만 유지
