# Diagram Guidelines

## 공통 원칙
- 비즈니스 흐름 다이어그램을 기술 흐름보다 먼저 배치한다.
- 다이어그램 수는 타입별 1개로 제한하지 않는다.
- 각 다이어그램에는 목적(어떤 의사결정을 지원하는지)을 1줄로 남긴다.

## C4 Container
- 시스템 경계, 외부 시스템, 저장소를 먼저 배치한다.
- 컨테이너 이름은 역할 기반으로 명확히 쓴다.

## C4 Component
- 컨테이너 내부 책임 분리를 보여준다.
- 의존 방향은 상위 -> 하위로 일관되게 유지한다.

## Sequence
- 핵심 시나리오를 먼저 작성하고, 비교/예외/복구가 필요하면 여러 개로 분리한다.
- 실패/예외 응답 경로를 반드시 포함한다.

## Flowchart
- 조건 분기와 종료 조건을 명확히 표현한다.
- 상태 전이 또는 처리 절차를 목적별로 분리해 여러 개 작성할 수 있다.
