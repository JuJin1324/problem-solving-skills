# Design Implementation 철학

## 왜 필요한가
- design-phase는 스프린트 설계이고, 구현 설계는 코드 전 단계의 기술 설계다.
- 구현 설계를 분리하면 코드 단계에서의 구조 오류를 줄일 수 있다.
- 재사용 가능한 라이브러리/서비스를 먼저 검토하면 구현 비용과 리스크를 줄일 수 있다.

## 원칙
- US 단위로 설계를 작성한다.
- Sequence와 Flowchart를 기본 세트로 사용한다.
- C4 Container는 외부 의존성이 복잡할 때 선택적으로 사용한다.
- 정상 흐름뿐 아니라 실패 흐름을 반드시 포함한다.
- 기능 목록과 ADR을 함께 기록한다.

## 완료의 의미
- 문서가 완성된 상태가 아니라, 구현자가 즉시 코드로 옮길 수 있는 상태를 의미한다.
