# US Complete

## 목적

US(User Story)를 완료할 때 배운 점과 의사결정을 기록하고, 커밋까지 일관된 흐름으로 마무리합니다.

**해결하는 문제:**
- US 완료 후 기록 없이 다음 US로 넘어감 → 배움과 의사결정 맥락 유실
- 커밋 타이밍을 놓쳐 작업 단위가 뒤섞임

**제공하는 가치:**
- US 단위 배움이 기록됨 → Sprint 회고 시 근거 있는 회고 가능
- 면접에서 "이 과정에서 무엇을 배웠는가"에 구체적으로 답변 가능
- 깔끔한 커밋 히스토리 유지

---

## 파일 구조

```
.agile/
└── sprints/
    └── sprint-N/
        ├── plan.md                 # 완료 태스크 [x] 체크
        ├── us-{N.M}-retrospective.md      # US별 회고 (이 스킬이 생성)
        └── sprint-retrospective.md        # Sprint 회고 (/sprint-complete)
```

---

## 작동 방식

### Step 1: 완료 확인

US의 모든 Step이 완료되었는지 plan.md를 확인한다.
- 이동/보류된 Step이 있으면 사유를 기록한다.

### Step 2: plan.md 업데이트

완료한 태스크를 `[ ]` → `[x]`로 체크한다.

### Step 3: US 회고 문서 작성

**파일 경로:** `.agile/sprints/sprint-N/us-{N.M}-retrospective.md`

**템플릿:**

```markdown
# US-{N.M} 회고: {US 제목}

📅 완료일: {날짜}

---

## 산출물

| 파일 | 역할 |
|------|------|
| `{경로}` | {설명} |

---

## 배운 점

### {배운 것 1}
- {설명 — 몰랐다가 알게 된 것, 예상과 달랐던 것}

### {배운 것 2}
- {설명}

---

## 의사결정

### {결정 제목}
- **선택:** {무엇을 선택했는가}
- **이유:** {왜 이 결정을 했는가}

---

## 아쉬운 점 / 개선할 점

- {이번 US에서 아쉬웠던 점, 다음에 바꿀 것}

---

## 다음 단계 연결

- {이 US의 결과가 다음 US/Sprint에 어떻게 연결되는가}
```

**작성 원칙:**
- "무엇을 했는가"보다 "무엇을 배웠는가 + 왜"에 집중
- 6개월 후에도 이해할 수 있도록 근거를 명시
- 짧아도 되지만 배운 점은 반드시 포함

### Step 4: 커밋 요청

회고 작성 후 사용자에게 커밋 진행 여부를 확인한다.

**커밋 메시지 형식:**
```
US-{N.M}: {US 제목 요약}

- {주요 변경 사항 1}
- {주요 변경 사항 2}
- {주요 변경 사항 3}
```

---

## 자동 트리거 조건

AI는 다음 상황에서 이 스킬을 **자동으로 실행**한다:

1. US의 마지막 Step 작업이 완료되었을 때
2. 사용자가 "US 끝났다", "다음으로 넘어가자" 등의 의사를 표현했을 때
3. 사용자가 `/us-complete`를 직접 호출했을 때

---

## 중요 원칙

### 1. US 단위 커밋

- 하나의 US = 하나의 커밋 (원칙)
- US 내 중간 커밋이 필요하면 사용자 판단에 따름

### 2. 회고는 배움 중심

- "산출물 목록"이 아닌 "왜 이렇게 결정했는가 + 무엇을 배웠는가"를 기록
- Sprint 회고 때 US 회고들을 집약하므로, US 회고는 간결해도 됨

### 3. 커밋 전 확인

- AI가 임의로 커밋하지 않음
- 반드시 사용자에게 커밋 여부를 확인
