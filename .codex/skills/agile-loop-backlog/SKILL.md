# Agile Loop Backlog

## 목적
2W를 바로 1H로 넘기지 않고, **백로그 정제와 MVP 컷라인**을 먼저 확정해 설계/실행 오차를 줄인다.

핵심 원칙:
- MVP를 1스프린트에 끝내는 것이 목표가 아니다.
- 대신 1스프린트 안에 검증 가능한 최소 단위(`MVP-0`)는 반드시 만든다.
- 전체 사용자 가치 전달판(`MVP-1`)은 여러 스프린트로 분할 가능하다.

## 이 스킬이 메인으로 담당하는 것
- 2W 기반 백로그 후보(에픽/US) 도출
- 수직 슬라이스 기준으로 US 분할
- 우선순위 점수화(가치/학습효과/노력)
- MVP-0, MVP-1 경계 확정
- 이번 스프린트 컷라인 확정(1H 전달 입력)

## 입력
- `problems/[문제명]/2w-vN.md`
- `problems/[문제명]/case-study-vN.md` (선택)
- `problems/[문제명]/backlog-vN-1.md` (선택: 기존 backlog가 있으면 필수 참조)

## 출력물
- `problems/[문제명]/backlog-vN.md`

템플릿:
- `templates/backlog-vN.md`

참고 문서:
- `references/philosophy.md`
  - MVP를 1스프린트로 끝내야 하는지 판단이 흔들릴 때
  - MVP-0 / MVP-1 분리 기준이 필요할 때

## 작동 방식

### 1단계. 2W + 이전 Backlog 입력 고정
2W에서 아래만 고정 입력으로 사용:
- 무엇/왜
- 성공 기준/경계
- 제약 조건

기존 `backlog-vN-1.md`가 있으면 아래를 먼저 확정:
- 유지할 Epic/US
- 변경할 Epic/US
- 제거할 Epic/US

### 2단계. 백로그 후보 도출
2W를 기반으로 후보를 도출:
- 에픽 1~3개
- US 3~8개
- 각 US에 사용자 가치 1줄, 검증 증적 1줄 기록

### 3단계. 수직 슬라이스 분해
US를 독립 검증 가능 단위로 분해:
- 의존성이 큰 항목은 더 작은 US로 재분할

### 4단계. 우선순위 점수화
각 US를 아래 3축으로 점수화:
- 가치(Value): 1~3
- 학습/리스크 감소(Learning): 1~3
- 노력(Effort): 1~3
- 우선순위 점수 = `(Value + Learning) / Effort`

### 5단계. MVP 컷라인 확정
아래 두 범위를 분리:
- MVP-0: 1스프린트 검증판 (US 1~2개)
- MVP-1: 사용자 가치 전달판 (복수 스프린트 가능)

MVP-0 적합성 게이트:
- 데모 가능한 End-to-End 흐름 1개 존재
- 측정 가능한 지표 1개 이상 존재
- 블로킹 의존성 3개 이하
- 고위험 미확정 항목 1개 이하

### 6단계. 1H 전달 입력 확정
`backlog-vN.md`에 아래를 확정:
- 이번 스프린트 대상 US(1H In 후보)
- 다음 라운드 US(Out/Later 후보)
- 미확정 항목(Unknown)

## 안티패턴
- MVP를 "전체 제품 1차 완성"으로 정의함
- 백로그를 기술 태스크 목록으로만 작성함
- 1스프린트 불가능한 큰 US를 유지함
- 2W에서 바로 1H로 넘어가 우선순위 결정을 생략함
- 이전 backlog를 무시하고 매번 처음부터 재작성함

## 완료 조건
- `backlog-vN.md` 작성 완료
- US별 우선순위 점수 확정
- MVP-0 / MVP-1 경계 확정
- 1H 전달용 In/Out/Unknown 확정
