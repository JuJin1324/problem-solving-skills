# US Complete

## 목적

US(User Story)를 완료할 때 산출물과 의사결정을 기록하고, 커밋까지 일관된 흐름으로 마무리합니다.

**해결하는 문제:**
- US 완료 후 기록 없이 다음 US로 넘어감 → 의사결정 맥락 유실
- 커밋 타이밍을 놓쳐 작업 단위가 뒤섞임

**제공하는 가치:**
- US 단위로 산출물과 의사결정이 기록됨
- 깔끔한 커밋 히스토리 유지

---

## 파일 구조

```
.agile/
└── sprints/
    └── sprint-N/
        ├── plan.md                 # 완료 태스크 [x] 체크
        ├── us-{N.M}-summary.md     # US별 summary (이 스킬이 생성)
        └── retrospective.md        # Sprint 회고 (/sprint-complete)
```

---

## 작동 방식

### Step 1: 완료 확인

US의 모든 Step이 완료되었는지 plan.md를 확인한다.
- 이동/보류된 Step이 있으면 사유를 기록한다.

### Step 2: plan.md 업데이트

완료한 태스크를 `[ ]` → `[x]`로 체크한다.

### Step 3: Summary 문서 작성

**파일 경로:** `.agile/sprints/sprint-N/us-{N.M}-summary.md`

**템플릿:**

```markdown
# US-{N.M} Summary: {US 제목}

📅 완료일: {날짜}

---

## 산출물

| 파일 | 역할 |
|---|---|
| `{경로}` | {설명} |

---

## 주요 의사결정

### 1. {의사결정 제목}

- {변경 내용}
- 근거: {왜 이 결정을 했는가}

---

## 다음 단계

- {이 US의 결과가 다음 US에 어떻게 연결되는가}
```

**작성 원칙:**
- 산출물은 실제 생성/수정된 파일 경로를 명시
- 의사결정은 "무엇을 → 왜" 형태로 근거를 반드시 포함
- 다음 단계는 후속 US와의 연결 고리를 명시

### Step 4: 커밋 요청

Summary 작성 후 사용자에게 커밋 진행 여부를 확인한다.

**커밋 메시지 형식:**
```
US-{N.M}: {US 제목 요약}

- {주요 변경 사항 1}
- {주요 변경 사항 2}
- {주요 변경 사항 3}
```

---

## 자동 트리거 조건

AI는 다음 상황에서 이 스킬을 **자동으로 실행**한다:

1. US의 마지막 Step 작업이 완료되었을 때
2. 사용자가 "US 끝났다", "다음으로 넘어가자" 등의 의사를 표현했을 때
3. 사용자가 `/us-complete`를 직접 호출했을 때

---

## 중요 원칙

### 1. US 단위 커밋

- 하나의 US = 하나의 커밋 (원칙)
- US 내 중간 커밋이 필요하면 사용자 판단에 따름

### 2. Summary는 미래의 나를 위한 기록

- "왜 이렇게 결정했는가"를 6개월 후에도 이해할 수 있도록 작성
- 과정이 아닌 결론과 근거를 기록

### 3. 커밋 전 확인

- AI가 임의로 커밋하지 않음
- 반드시 사용자에게 커밋 여부를 확인
